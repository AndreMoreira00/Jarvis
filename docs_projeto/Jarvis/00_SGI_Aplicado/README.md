# 00 · SGI Aplicado

Pasta onde se documenta como normas, procedimentos e gates de qualidade se aplicam **a este projeto**.

> Nesta fase o Jarvis e um **prototipo de software** (sem hardware fisico, firmware, producao ou certificacoes). Por isso o SGI aplicado e **leve**: nao ha auditorias, gates formais, nao conformidades nem procedimentos regulatorios em vigor. O foco atual e apenas registrar **objetivos** e **riscos** do projeto, que guiam as decisoes de engenharia. As demais subpastas (auditorias, NCs, gates, registros) ficam vazias ate que o projeto evolua para hardware/producao.
